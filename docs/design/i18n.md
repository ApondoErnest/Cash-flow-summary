# UI Localization (English / French)

[← Documentation hub](../README.md) | [ux-overview.md](ux-overview.md) | [csv-specification.md](csv-specification.md)

Cash Flow Summary is **fully bilingual** in the user interface: **English (`en`)** and **French (`fr`)**.

CSV import language is handled separately — files are auto-detected from headers (see [csv-specification.md](csv-specification.md)); users do not pick CSV language manually.

---

## Supported locales

| Code | Language | Notes |
|------|----------|-------|
| `en` | English | Default when no preference is stored |
| `fr` | French | Organization default in seeded data |

Configuration: [`config/locale.php`](../../config/locale.php), [`config/app.php`](../../config/app.php) (`APP_LOCALE`, `APP_FALLBACK_LOCALE`).

---

## User preference

### At login

The login form includes a **Language** selector (`en` / `fr`). The chosen locale is stored in session and a long-lived cookie before redirect.

### After login

The app header includes an **EN | FR** toggle (`LanguageSwitcher` Livewire component). Switching updates session + cookie immediately and re-renders the page in the selected language.

### Resolution order

1. Session key `locale` (explicit user choice)
2. Cookie `locale`
3. `APP_LOCALE` from environment / config

## Implementation conventions

| Area | Convention |
|------|------------|
| User-visible strings | Laravel `__('file.key')` — no hardcoded UI copy in Blade/PHP |
| Translation files | `lang/en/*.php`, `lang/fr/*.php` |
| Navigation labels | `lang/*/navigation.php` via `RoleNavigation` |
| Page titles (placeholders) | `lang/*/pages.php` keyed by route name |
| Roles | `lang/*/roles.php` via `UserRole::label()` |
| Middleware | `App\Http\Middleware\SetLocale` on web stack |
| Helper | `App\Support\Locale\AppLocale` |
| HTML `lang` | BCP 47 via `AppLocale::htmlLang()` (`fr-CM` / `en-GB`) |
| Dates & times | Custom `x-ui.date-picker` / `x-ui.time-picker`; FR = 24h (`HH:MM`), EN = 12h (`h:mm AM/PM`); storage always `H:i` |
| Money amounts | `DashboardMoney::format()` — FR `1 202 130,00`, EN `1,202,130.00`; WhatsApp uses org preferred language |
| Carbon | `Carbon::setLocale()` applied with UI locale |

Native `<input type="time">` / `<input type="date">` are **not** used for UI — browsers ignore app language and often stay English (AM/PM).

When adding new UI (forms, nav items, flash messages, validation copy):

1. Add keys to **both** `lang/en/` and `lang/fr/`.
2. Use `__()` in Blade and PHP.
3. Add Pest coverage for at least one French string on new screens.
